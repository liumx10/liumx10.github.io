My past reserach focus on the following aspects

  • Cooperative task management for building distributed transactions

Many transactional systems consists of many waiting points, including both disk/network IOs and algorithm-level ones.
There are two major models to solve this problem: multi-thread model, in which OS will automatically switch to another thread if one thread is blocked by waiting points.
Context switch will be a high overhead in this model; event-driven model, in which every waiting point generates a event and a callback function for the event.
Even-driven has no context switch, but it leads to the “stacking-ripping” problem which incurs tedious code readability and maintainability issues.
We provided SSEvent, an event-driven framework that helps eliminate the stacking-ripping problem.
